Request Data Flow

Example: User selects "Q2-2025" + "London" on Dashboard

1
User Interaction
FilterBar dropdowns update shared refs in useFilters.js
selectedPeriod.value = 'Q2-2025' selectedLocation.value = 'London'
2
Vue Watcher Fires
Each view watches all 4 filter refs and calls loadData() independently
watch([selectedPeriod, selectedLocation, selectedCategory, selectedStatus], () => loadData())
3
API Call via Axios
getCurrentFilters() converts refs to query params, omitting "all" values
GET http://localhost:8001/api/dashboard/summary?warehouse=London&month=Q2-2025
4
FastAPI Filtering
apply_filters() narrows by warehouse; filter_by_month() expands quarter to months
QUARTER_MAP['Q2-2025'] → ['2025-04', '2025-05', '2025-06'] orders filtered to: London + Apr/May/Jun
5
JSON Response
Pydantic validates the response shape before serialization
{ total_inventory_value, low_stock_items, pending_orders, total_backlog_items, total_orders_value }
6
Vue Reactive Update
Refs update, computed properties recalculate, charts and tables re-render declaratively — no manual DOM manipulation
summary.value = response.data // computed: statusData, topProducts, categoryData → re-evaluate automatically

Vue Composables
useFilters.js
Singleton refs for all 4 global filters. getCurrentFilters() converts to API params.
useAuth.js
Current user state, profile info, session handling.
useI18n.js
t() translation fn, currentCurrency ref, EN/JA + USD/JPY switching.
Key Frontend Patterns
Watcher Pattern
Every view watches all 4 filters, calls loadData() on any change.
Computed Chain
Raw data in refs; all derived values (charts, KPIs, tables) in computed properties.
Custom SVG Charts
No chart library. Donut via stroke-dasharray; bars via dynamic width %.
Modal System
7 modals use :is-open + @close pattern. Parent holds state, passes data via props.
Stock Sorting
Inventory sorted by status order: lowStock → adequate → inStock.
API Proxy
api.js centralizes all Axios calls. CORS: backend allows all origins in dev.

Data Files — server/data/

📄
inventory.json
~100 items. SKU, name, category, warehouse, qty_on_hand, reorder_point, unit_cost, location.
📄
orders.json
~200 orders for 2025 (Jan–Dec). Status, warehouse, category, order_date, total_value.
📄
demand_forecasts.json
~30 forecasts. current vs forecasted demand, trend (Increasing / Stable / Decreasing).
📄
backlog_items.json
~10 shortage items. qty_needed vs qty_available, days_delayed, priority (high/med/low).
📄
spending.json
Pre-aggregated spend summary + monthly breakdown + category spend. Not dynamically filtered.
📄
transactions.json
~150 individual transactions. Date, type, amount, category, vendor, status (approved/pending).
📄
purchase_orders.json
Starts empty. Written at runtime by POST /api/purchase-orders. Persisted in-process only.
